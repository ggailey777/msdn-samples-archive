Introduction

This Windows Store sample application supports the Adding a mobile service (XAML and C#/VB or HTML/JavaScript) topics in the Windows Dev Center for Windows Store apps. This topic shows you how to use Visual Studio 2013 to connect to Azure Mobile Services for data storage in a Windows Store app. The Windows Store app in this Visual Studio 2013 project initially stores inserted data in an in-memory collection. By following the tutorial, you replace the in-memory collection with Mobile Services for data storage and access. Subsequent tutorials further modify this project to send push notifications by using Mobile Services.

Prerequisites

  • Microsoft Visual Studio 2013.
  • Active Microsoft Azure account.
    If you don't have an account, you can create a free trial account in just a couple of minutes. For details, see Microsoft Azure Free Trial.
  • This GetStartedWithMobileServices project, a Windows Store app that will be modified to leverage Mobile Services for data storage.

Building and running the initial sample

  1. In Visual Studio 2013, open the sample solution GetStartedWithData.
  2. Press F5 to build and start this sample.
  3. Enter text in Insert a TodoItem and click Save.
    Added items appear as a checkbox item in a list in the second column, along with the assigned item ID.
  4. Stop debugging, and then press F5 to run the app again.
    Notice that the previously added items are no longer there, because they were only stored in memory.

Updating the app to leverage Mobile Services

To enable the app to leverage Mobile Services for cloud-based data storage instead of local or in-memory storage, complete the Quickstart: Adding a mobile service (XAML and C#/VB or HTML/JavaScript). This topic walk you through the following steps.

  1. Download and run the project.
  2. Create a new mobile service.
  3. Add a new table for data storage.
  4. Update the app to use Mobile Services.
  5. Test the app against the new mobile service.
  6. Modify the query to filter out completed items.

Note that once you have an active Windows Azure subscription, all of these steps in these tutorials are performed in the Visual Studio 2013 development environment. Once you have updated the app, it uses Mobile Services to store and retrieve data instead of using local or in-memory storage.

Running the app against Mobile Services

  1. Press F5 to start the app.
  2. Enter text in Insert a TodoItem and click Save.
  3. Stop debugging, and then press F5 to run the app again.
    Notice that the  previously added items are restored after the app loads.
  4. (Optional) Using an HTTP message inspection program, like Fiddler, repeat Step 2 and review the insert request sent to the mobile service, then click Refresh and again review the query request that is sent to the mobile service.

More Information
