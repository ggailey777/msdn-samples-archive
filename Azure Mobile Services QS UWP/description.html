Introduction

This universal Windows 8.1 app project sample shows you how to use Azure Mobile Services to leverage data in a universal Windows app project for both Windows Store and Windows Phone Store apps. The app project uses a shared view-model class to initially store inserted data in an in-memory collection. By following the tutorial, you replace the in-memory collection in the view-model with Mobile Services for data storage and access.

Prerequisites

Completing the project requires the following:

  • Visual Studio 2013 Update 2, which supports universal Windows app projects.
  • An active Microsoft Azure account.

Building and running the initial universal Windows app

  1. In Visual Studio 2013 Update 2, open the sample solution GetStartedWithData.
  2. Press F5 to build and start the Windows Store 8.1 app.
  3. Enter text in the textbox and click Save.
    Added items appear as a checkbox item in the list.
  4. Stop debugging, and then press F5 to run the app again.
    Notice that the previously added items are no longer there, because they were only stored in memory.
  5. Right-click the Windows Phone 8.1 project, click Set as Start-up Project, then press F5 to start the Windows Phone Store app.
  6. Repeat steps 3 and 4 to verify that the sample behaves the same way.
    The shared view-model is used by each project for data binding to the in-memory collection. 

Updating the app to leverage Mobile Services

To enable the app to leverage Mobile Services for cloud-based data storage instead of local or in-memory storage, you must complete the following steps:

  1. Download this universal Windows 8.1 app solution
  2. Create the mobile service
  3. Add a data table for storage
  4. Update this app to use Mobile Services (following the code comments in the shared code files)
  5. Test the app against the live mobile service

Once you have updated the app, it will use Mobile Services to store and retrieve data instead of using local or in-memory storage. Both apps will have access to the same stored data.

Running the app against Mobile Services

  1. Press F5 to start the app.
  2. Enter text in the textbox and click Save.
  3. Stop debugging, and then press F5 to run the app again.
    Notice that the  previously added items are restored after the app loads.
  4. Switch the start-up project and verify that the same stored data is displayed in the other version of the app.
