Introduction

This sample contains the Visual Studio projects that are created when you complete the WCF Data Services quickstart in the WCF Data Services 5.0 product documentation. The quickstart helps you become familiar with the Open Data protocol (OData) and WCF Data Services through a series of tasks that support the topics in Getting Started with WCF Data Services.

Note: WCF Data Services 5.0 supports OData v3. However, this quickstart does not demonstrate any of the functionality that was introduced in OData v3. For more information, see What's New in WCF Data Services.

Prerequisites

To use this sample, you must install the following components:

  • WCF Data Services 5.0.  
    Note: This product version is available outside of the .NET Framework. The quickstart itself is supported on all versions of WCF Data Services and OData. However, this sample requires WCF Data Services 5.0 to run.
  • Visual Studio 2010, including Entity Data Model tools and Entity Framework.
  • The default instance of SQL Server Express.
    Note: You can use any instance of SQL Server to host the Northwind database, but you must modify the connection string in the data service web.config file so that the service will connect to the correct Northwind instance. If there are runtime errors attaching the Northwind.mdb database file provided, give your user account full rights to the physical file in the app_data folder.

WCF Data Services Quickstart Tasks

The following quickstart tasks, when completed, create the sample client projects and the writable Northwind data service included in this sample.

Accessing OData Feeds from a Web Browser
Access the read-only Northwind OData service by submitting HTTP GET requests through a Web browser to the public feed.
Consuming OData Feeds in a Console Application
Create a WPF client application to consume the public Northwind OData feed and bind returned feed data to Windows controls.
Creating the Northwind Data Service
Define ASP.NET application, define the data model, create the data service, and enable access to resources.
Consuming OData Feeds in a Console Application
Create a second client application to access the new read-write Northwind service. This new client adds functionality to bind data to controls and send changes back to the data service.
