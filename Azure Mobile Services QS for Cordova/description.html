Introduction

An Azure Mobile Services .NET backend project supports HTML apps, including Apache Cordova app projects. However, there isn't currently a downloadable quickstart project for Cordova apps with a .NET backend mobile service. This sample project contains both a .NET backend mobile service, which supports the TodoList quickstart project, but it also has an Apache Cordova for Visual Studio project that connects to the .NET backend (when published to Azure).

Prerequisites

Completing the project requires the following:

  • Visual Studio Community 2013, which includes the Apache Cordova tools
  • An active Microsoft Azure account.
  • A .NET backend mobile service. To see how to create a new .NET backend mobile service, see the section Create a new mobile service.

Configure and publish the mobile service project

Once you have created your .NET backend mobile service as shown in the section Create a new mobile service, you must configure the project in this sample to connect to your mobile service.

  1. In the todolist_cordova_dotnetService project in Visual Studio, open the Web.config file and replace the MS_MobileServiceName app setting with the name of your .NET backend mobile service.
  2. Publish the project from Visual Studio, as described in the section Publish the mobile service to Azure.

Next, you will configure the Cordova app project to access the mobile service you just published.

Configure and test the Cordova app

  1. In Visual Studio, expand the Todolist_Cordova_Client project and open the index.js file in the \scripts folder.
  2. Update the serviceUrl and service AppKey variables with the mobile service URL and application key, respectively. You can find this information in the Azure portal on the Dashboard tab for your mobile service. 
  3. Select your client platform and press F5.
  4. After the is loaded, enter text in the textbox and click Save.
    Added items appear as a checkbox item in the list.
  5. Stop debugging, and then press F5 to run the app again.
    You should see the same data, which was persisted in Azure.

You cannot connect a Cordova app to a local instance of a .NET backend mobile service, unless you first configure SSL on the local web site. The Cordova client requires SSL even for localhost.

For more information about Mobile Services and cross-platform apps, see Supporting multiple device platforms from a single mobile service.
