Introduction

The Open Data Protocol (OData) is based on an entity and relationship model that enables you to access data in the style of representational state transfer (REST) resources. OData enables the use of the standard HTTP protocol to execute queries, and even to create, update, and delete data from a remote data service. By using the OData client library for Windows Phone, your Windows Phone application can consume data from a Windows Azure service or any other service that supports the OData protocol. This sample supports the topic Consuming a Windows Azure Data Service by using the OData Client.

Important: This sample project has been updated to use the improved support for OData in Windows Phone 7.5 that is included in the Windows Phone SDK 7.1. The prerequisites for this project have been updated.

Features

This sample uses the following features of the OData client library for Windows Phone.

  1. Data binding
  2. Tombstoning
  3. View-model programming pattern
  4. Client-requested paging
  5. Support for server-side paging
  6. Using the Add Service Reference tool to generate the proxy classes
  7. Requesting the media resource stream URI

Prerequisites

Before running this sample, make sure that the following software is installed:

  • Visual Studio 2010
  • Windows Phone SDK 7.1

Building and Running the Sample

Use the following procedures to build and run the sample.

To update the reference and build the sample

  1. Unzip the sample files on the local computer.

  2. In Visual Studio 2010, open the solution file ODataWinPhoneQuickstart.sln.

  3. Build the project.

To run the application

  1. Press F5.

    This deploys the application to the Windows Phone emulator and runs the sample in debug mode.

  2. Click Next Page to load the next client-requested page of results, click a title to see details about the title, or click the Back button to return to the previous page.

  3. Click the Start button.

    This causes the application to deactivate and initiates tombstoning.

  4. Click the Back button.

    This causes the application to reactivate and application state is restored.
